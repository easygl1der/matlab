## MATLAB多项式表示与运算

□ MATLAB习惯将降幂排列的多项式 $a(x)=a_{1} x^{n}+a_{2} x^{n-1}+ \ldots+a_{n} x+a_{n+1}$ 存储为系数行向量 $a=\left[a_{1}, a_{2}, \ldots, a_{n+1}\right]$
□ 多项式的乘法，基于MATLAB的多项式系数定义，有：

$$
\begin{aligned}
& \sum_{i=1}^{m+1} a_{i} x^{m+1-i} \cdot \sum_{j=1}^{n+1} b_{j} x^{n+1-j} \\
& =\sum_{k=2}^{m+n+2}\left(\sum_{i=\max (1, k-n-1)}^{\min (m+1, k-1)} a_{i} b_{k-i}\right) x^{m+n+2-k}
\end{aligned}
$$

注意到红字，即乘积多项式的系数恰为两个原始系数的卷积 $\square \mathrm{c}=\mathrm{conv}(\mathrm{a}, \mathrm{b})$ 就是一种快速获取多项式相乘系数的方法 $\square[q, r]=$ deconv（ $b, a$ ）为多项式带余除法，$q$ 为商 $r$ 为余式 $\square[r, p, k]=$ residue $(b, a)$ 即表示 $\frac{\boldsymbol{b}(\boldsymbol{x})}{\boldsymbol{a}(\boldsymbol{x})}=\sum_{\boldsymbol{i}=\mathbf{1}}^{\boldsymbol{n}} \frac{\boldsymbol{r}_{\boldsymbol{i}}}{\boldsymbol{x}-\boldsymbol{p}_{\boldsymbol{i}}}+\boldsymbol{k}(\boldsymbol{x})$ ，其中 $r$ 为留数向量，$p$ 为极点向量，$k$ 为余式系数行向量

## MATLAB有限长向量卷积函数conv

□已知 $a=\left[a_{1}, a_{2}, \ldots, a_{n}\right], b=\left[b_{1}, b_{2}, \ldots, b_{m}\right]$ ，类似于刚才多项式乘法的例子，conv $(\mathrm{a}, \mathrm{b})$ 函数将返回 $\boldsymbol{a} * \boldsymbol{b}$ 的向量，即

$$
(a * b)_{k}=\sum_{i=1}^{k-1} a_{i} b_{k-i}, 2 \leq k \leq m+n
$$

这里补充定义 $\forall i>n, a_{i}=0, \forall j>m, b_{j}=0$
$\square \operatorname{conv}(\mathrm{a}, \mathrm{b}$, ＇same＇$)$ 将返回一个长度等于 $\boldsymbol{n}$ 的卷积结果，即截取上述卷积结果中间 $\boldsymbol{n}$ 个元素的结果，常用于信号或图像处理问题。若 3 个选 2 个，则默认选择后两个。
$\square \operatorname{conv}\left(\mathrm{a}, \mathrm{b},{ }^{\prime}{ }^{\prime} \mathrm{valid}^{\prime}\right)$ 将返回一个长度等于 $\boldsymbol{n}-\boldsymbol{m}+\mathbf{1}$ 的卷积结果，即仅包含 $b$ 的全部元素参与卷积的中间项。
$\square$ deconv 为反卷积函数。卷积运算之前对于边界条件的处理有时需要跟实际问题相结合。具体方法有 0 填充（默认），对称延拓，复制延拓，周期延拓等。

## 动态规划例 1 －最长不降子序列问题

设有有限长度的数列 $a_{1}, a_{2}, \ldots, a_{n}$ ，若存在子列 $a_{i_{1}}, a_{i_{2}}, \ldots a_{i_{k}}$ ，使得 $\forall \boldsymbol{m}<\boldsymbol{n}, \boldsymbol{a}_{\boldsymbol{i}_{\boldsymbol{m}}} \leq \boldsymbol{a}_{\boldsymbol{i}_{\boldsymbol{n}}}$ ，称子列为不降子序列。本问题的目的为计算 $a_{1}, a_{2}, \ldots, a_{n}$ 的最长不降子序列长度。

口如数列 $13,7,9,5,16,38,24,37,18,44,19,21,22,63,15$ 。很容易获取一个不降子序列 $13,16,38,44,63$ ，长度为 5 ，但这个并不是最长不降子序列。但该数列的最长不降子序列为 $7,9,16$ ， $18,19,21,22,63$ ，长度为 8 。如果获取该子列呢？
－暴力枚举法：（算法复杂度 $O\left(2^{n}\right)$ ，可以确保获得正确答案，但当 $\boldsymbol{n}$ 较大时，会出现过长的时间消耗，导致算法崩溃）
□ 各种比较直接的快速算法（都不对）：如1．从最小的数开始，此方法从 5 开始，只能得到长度为 7 的不降子序列。2．分治法，以 37 为中点，左侧最优解 $7,9,16,24,37$ ，右侧最优解 $18,19,21,22,63$ ，总长度达到 10 ，但两组解并不能衔接。

## 动态规划－保证正确的算法精简方式

－动态规划的两大核心要件：最优子结构与重叠子问题。利用最优子结构的性质，减少暴力枚举中重复的运算与比较。

考虑数列 $a_{1}, a_{2}, \ldots, a_{k}, a_{k+1}$ 的最长不降子序列问题。从最后一个元素出发，可以有两种选择。分析如下：
1）最长不降子序列不包含 $a_{k+1}$ ，此时将 $a_{1}, a_{2}, \ldots, a_{k}$ 的最长不降子序列结果援引过来即可。
2）最长不降子序列包含 $\boldsymbol{a}_{\boldsymbol{k}+\boldsymbol{+}}$ ，此时需确定其上一项 $\boldsymbol{a}_{\boldsymbol{j}}, \mathbf{1} \leq \boldsymbol{j} \leq n$ ，在满足前提 $a_{j} \leq a_{k+1}$ 的前提下，新序列长度可以等于 （ $a_{1}, a_{2}, \ldots a_{j}$ 中包含 $a_{j}$ 的最长不降子序列长度）+1 。
3）比较上述所有情况，获得 $a_{1}, a_{2}, \ldots, a_{k}, a_{k+1}$ 的最长不降子序列的长度。
－我们发现最关键的递推式比较依赖于 $a_{1}, a_{2}, \ldots a_{j}$ 之中包含 $a_{j}$的最长不降子序列长度。我们记这个长度为 $\boldsymbol{F}(\boldsymbol{j})$ 。

## 动态规划－保证正确的算法精简方式

－对于本问题而言，最优子结构就是每一个 $F(j)$ 的值。因为计算 $F(k+1)$ 可以利用之前计算的 $F(1), F(2), \ldots F(k)$ 的结果，从而避免这些子问题的重复计算。

口动态规划的递推式也叫作状态转移方程。对本问题而言，状态转移方程可以记为：

$$
F(k+1)=\left\{\begin{array}{cc}
1, & k=0 \\
\max _{\substack{1 \leq j \leq k \\
a_{j} \leq a_{k+1}}} F(j)+1, k>0
\end{array}\right.
$$

此外，若后续 $F(k+1)$ 无法搜索到合适结果，也设 $F(k+1)=1$ 。
－因为最终获取的最长不降子序列未必包含最后一项，实际上的最长不降子序列长度应该等于 $\max _{1 \leq \boldsymbol{k} \leq \boldsymbol{n}} \boldsymbol{F}(\boldsymbol{k})$ 。
□ 因此，我们得到了一个算法复杂度为 $\boldsymbol{O}\left(\boldsymbol{n}^{2}\right)$ 的动态规划算法。

## 动态规划的算法代码

```
a = [13,7,9,5,16,38,24,37,18,44,19,21,22,63,15];
n = length(a);
F = ones(1, n); %设立状态转移函数,代表包含末项的最长不降子序列长度
for i = 2:n
    temp = F(1,1:i-1); %提取之前的状态转移函数
    ind = a(1,1:i-1)<=a(i); %考虑符合不减条件的上一项
    temp = ind.*temp; %将不符合条件的转移函数暂乘为0
    F(i) = max(temp) + 1; %新的状态转移函数由此生成
end
Result = max (F) %最大状态转移函数提供了问题的解
```

## 动态规划例2－矩阵连乘问题

□计算矩阵乘法 $A_{m \times n} B_{n \times p}$ ，一共需要得到 $m \times p$ 个元素，每个元素需要进行 $\boldsymbol{n}$ 次乘法和 $(\boldsymbol{n - 1})$ 次加法。因而算法消耗可以近似的用 $m n p$ 来表示。
□ 考虑实例 $A_{5 \times 1} B_{1 \times 5} C_{5 \times 1}$ 这样的连乘情况，先算 $A B$ ，再算 $A B C$ ，算法消耗总计为 $\mathbf{5} \times \mathbf{1} \times \mathbf{5}+\mathbf{5} \times \mathbf{5} \times \mathbf{1}=\mathbf{5 0}$ 。而根据矩阵乘法的结合律，先算 $B C$ ，再算 $A B C$ 也可以得到正确答案，算法总消耗为 $\mathbf{1} \times \mathbf{5} \times \mathbf{1}+\mathbf{5} \times \mathbf{1} \times \mathbf{1}=\mathbf{1 0}$ ，明显计算量更小。
－现假设有矩阵 $\boldsymbol{A}_{\mathbf{1}} \boldsymbol{A}_{\mathbf{2}} \cdots \boldsymbol{A}_{\boldsymbol{n}}$ 这样的连乘问题，根据矩阵乘法的合理性，需要有行（列）数指标共 $(n+1)$ 个，记为 $a_{1}, a_{2}, \ldots a_{n}$ ，求使用最佳顺序的情况下的最小算法消耗。
□ 在暴力枚举的体系下，每一次都可以在任意两个矩阵的间隔中进行＂抢先计算＂，容易计算出枚举的情况共有 $(\boldsymbol{n - 1})$ ！种，算法复杂度 $O(n!)$ 比较高。

## 矩阵连乘问题的子结构分解与递推

－更换思路，将中间某个位置的乘法看作最后一步，即 $\left(A_{1} A_{2} \cdots A_{i}\right)\left(A_{i+1} A_{i+2} \cdots A_{n}\right)$ 。在这种情况下，可以看成两更短的矩阵连乘问题，最后加上末步的计算代价 $a_{1} a_{i+1} a_{n}$ 。不过，由于 $1 \leq i \leq n-1$ ，我们需要考虑这 $(n-1)$ 种不同分割的结果，进行比较，得到最小的计算代价。
本问题的重复子问题与最优子结构就存在于一部分连续矩阵连乘的最小计算消耗。定义 $F(i, j), 1 \leq i \leq j \leq n$ 为 $A_{i} A_{i+1} \cdots A_{j}$的最小计算消耗。我们可以得到递推式

$$
F(i, j)=\left\{\begin{array}{cc}
0, & j=i \\
\min _{i \leq k<j} F(i, k)+F(k+1, j)+a_{i} a_{k+1} a_{j+1}, j>i
\end{array}\right.
$$

－容易发现，上述算法通过 2 个矩阵连乘计算消耗，递推到 3 个矩阵连乘， 4 个矩阵连乘⋯⋯，是一种金字塔式的递推结构。该算法的复杂度可以控制到 $O\left(n^{3}\right)$ ，远远超过暴力枚举。

## 动态规划的算法代码

```
a = [5,1,5,1,5,1,5,1];
n = length(a)-1;
F = zeros(n); %状态转移方阵
for len = 2:n %连乘矩阵的个数
    for i = 1:n-len+1 %连乘左端,此循环顺序至关重要
        j = i + len - 1;
        k = i:j-1; %批量化运算,减少一层for循环
        temp = F(i,k) + F(k+1,j)'+a(i)*a(k+1)*a(j+1);
        %批量计算一组状态转移函数,注意向量类型要一致
        F(i,j) = min(temp); %取最小值
    end
end
Result = F(1,n)
Result =
    22
```


## 动态规划例3－购物问题（0－1背包问题）

口假设某商店共有 $\boldsymbol{n}$ 件不同的商品，每件商品的价格是 $\boldsymbol{c}_{\boldsymbol{i}}$ 元，且 $c_{i} \in \mathbb{N}_{+}$，每件商品的性能值为 $\boldsymbol{v}_{i}$ 元。现你手上共有 $C$ 元（ $C \leq$ 2000），问如何购买可以使你购买物品的总性能值最大？
（每件物品都可以选择买或不买但都仅有 1 件， $\boldsymbol{v}_{\boldsymbol{i}}>\mathbf{0}$ ）
口该问题的另一个版本称为背包问题，而且对每一件物品只有不买（0）和买（1）两种选择，因此该问题称为 $0-1$ 背包问题。该问题的数学模型可以解释为一种整数规划问题：

$$
\begin{aligned}
& \max _{\left\{\boldsymbol{w}_{\boldsymbol{i}}\right\} \in\{0,1\}} \boldsymbol{w}_{\boldsymbol{i}} \boldsymbol{v}_{\boldsymbol{i}} \\
& \text { s.t. } \quad \boldsymbol{w}_{\boldsymbol{i}} \boldsymbol{c}_{\boldsymbol{i}} \leq C
\end{aligned}
$$

－暴力枚举所有情况，需要分别讨论每一件商品是否购买，并根据所有情况分析总价格和总性能值。算法复杂度达到 $O\left(2^{n} \cdot n\right), n$ 较大时，算法几近崩溃。根据＂性价比＂排序也同样存在错误。（高性价比物品有可能会导致手上现金资源应用不充分，如 $\vec{c}=[7,5,4], \vec{v}=[8,5,4], C=10$ ）

## 购物问题的最优子结构与递推

$\square$ 本题目最大的突破□在于 $\boldsymbol{c}_{\boldsymbol{i}}, C$ 为整数且规模有限。因此利用不同的总价格下的最优购买总性能值构造状态转移方程，是本题的突破点。因此我们设状态函数 $\boldsymbol{F}(\boldsymbol{i}, \boldsymbol{j})$ 为使用不超过 $j$ 元在前 $\boldsymbol{i}$ 件商品中选购时可以获得的最大性能值，其中 $\mathbf{0} \leq \boldsymbol{j} \leq \boldsymbol{C}$ 。

口如果前 $i$ 件物品用不超过 $j$ 元购买的最优购买组合为 $\left\{k_{1}, k_{2}, \ldots k_{l}\right\}$ 且 $k_{l}=i$（即第 $i$ 件需购买），则类似于最长不降子序列问题可以发现，前 $(i-1)$ 件物品用不超过 $\left(j-c_{i}\right)$ 元购买的最优组合必包含 $\left\{k_{1}, k_{2}, \ldots k_{l-1}\right\}$ ，否则就会发生矛盾。
－从而，当我们计算 $F(i, j)$ 并考虑第 $i$ 件物品是否购买时，可以比较 $F\left(i-1, j-c_{i}\right)+v_{i}$ 与 $F(i-1, j)$ 的大小，对比该物品购买或不购买的最优情况进行＂二选一＂。状态转移公式为：

$$
F(i, j)=\left\{\begin{array}{cc}
\max \left(F\left(i-1, j-c_{i}\right)+v_{i}, F(i-1, j)\right), & j \geq c_{i} \\
F(i-1, j), & j<c_{i}
\end{array}\right.
$$

上适用于 $i>1$ ，而 $i=1$ 的初始化比较简单，总复杂度 $O(n C)$ 。

## 动态规划的算法代码

```
c = [1,3,3,4,6,8,9,11,13];
v = [1,2,3,5,6,10,9,10,15];
C = 20;
n = length(c);
F = zeros(n,C+1); %因为j指标从0开始,需要多提供一列
F(1,1:C(1))=0; %买不起第一件商品则无性能值
F(1,C(1)+1:C+1)=v(1); %买得起第一件则获得第一件的性能值
```

for i = 2:n
$F(i, 1: c(i))=F(i-1,1: c(i))$; \%买不起第 $i$ 件则无需比较
F(i,c(i)+1:C+1) = max(F(i-1,1:C-c(i)+1) + v(i),F(i-
1, c(i)+1:C+1)) ;
\％当价格买得起时，买得起第 i 件则对比买与不买谁更优，取最大性能值 end

Result $=\mathrm{F}(\mathrm{n}, \mathrm{C}+1)$ \％最终结果对应 n 件产品遍历后 C 元的最大性能值
Result＝
23
